## 1. Objetivo

Este volume consolida todas as regras de negócio do **[NOME DO SISTEMA / PROJETO]**, extraídas dos documentos de requisitos, especificações funcionais, normas institucionais e demais artefatos aplicáveis.

Cada regra deverá possuir:
- identificador único;
- descrição formal;
- criticidade;
- origem documental;
- entidades afetadas.

---

## 2. Convenções

| Campo | Descrição |
|---|---|
| **ID** | Identificador único da regra no formato **RN-[DOMÍNIO]-[NUM]** |
| **Descrição** | Enunciado formal da regra de negócio |
| **Criticidade** | Classificação da relevância da regra (ex.: Alta, Média, Baixa) |
| **Origem** | Documento(s), seção(ões) ou requisito(s) de origem |
| **Entidades** | Entidades, objetos ou módulos afetados pela regra |
| **Observações** *(opcional)* | Restrições adicionais, exceções, dependências ou notas de implementação |

### 2.1 Padrão de Identificação

Adotar o seguinte padrão para codificação das regras:

- **RN-SEG** = Segurança e Acesso  
- **RN-CAD** = Cadastros Estruturantes  
- **RN-PLN** = Planejamento  
- **RN-EXE** = Execução  
- **RN-RES** = Resultados / Achados  
- **RN-REL** = Relatórios  
- **RN-MON** = Monitoramento  
- **RN-RSK** = Gestão de Riscos  
- **RN-INT** = Integridade e Rastreabilidade  
- **RN-CD** = Regras Transversais / Cross-Domain  

> Ajuste os prefixos conforme os domínios reais do projeto.
